Angular Component Lifecycle

Lifecycle Phases

Phase Hook Description
Creation constructor Invoked when the component is instantiated.
Change Detection ngOnChanges Called when input properties change.
ngOnInit Called once after the component is initialized.
ngDoCheck Invoked during change detection to check custom changes.
ngAfterContentInit Called after Angular projects external content into the component.
ngAfterContentChecked Called after checking the projected content for changes.
ngAfterViewInit Called after initializing the component's view and child views.
ngAfterViewChecked Called after checking the component's view and child views.
Destruction ngOnDestroy Called just before the component is destroyed.

Lifecycle Hooks Overview

  • ngOnChanges: Detects changes in input properties.
  • ngOnInit: Initializes the component after Angular sets its inputs.
  • ngDoCheck: Implements custom change-detection logic.
  • ngAfterContentInit: Runs after content projection is complete.
  • ngAfterContentChecked: Runs after projected content is checked.
  • ngAfterViewInit: Runs after the component's view is initialized.
  • ngAfterViewChecked: Runs after the view and its child views are checked.
  • ngOnDestroy: Performs cleanup just before component destruction.
